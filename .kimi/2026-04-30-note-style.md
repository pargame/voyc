# Note Writing Style (2026-04-30)

`voyc/notes/` 내 문서를 작성할 때의 스타일과 범위 규칙을 정리합니다.

## 노트 분류와 범위

| 분류 | 파일 패턴 | 작성 범위 |
|------|-----------|-----------|
| **언어/기술 일반** | `cpp_*.md`, `STL_*.md` | 보편적 지식. 프로젝트 맥락(변수명, 경로)을 넣지 않는다. |
| **툴/환경** | `gh_basics.md`, `zsh_basics.md` | 해당 도구의 일반 사용법. 프로젝트 예시는 최소화한다. |
| **프로젝트 전용** | `build-toolchain.md`, `vscode-configs.md` | voyc의 설정과 환경만 다룬다. |

## cpp/STL 노트 작성 원칙

- **변수명**: `src`, `result`, `file` 같은 일반적인 이름 사용. `voyc::lexSource` 같은 프로젝트 특화 이름은 금지.
- **경로**: `/Users/...` 같은 절대 경로 금지. 필요시 `./build/voyc`만 예외 허용.
- **코드 예제**: 표준 라이브러리만 사용. 프로젝트 헤더 포함 금지.
- **설명**: "이 프로젝트에서는" 금지. "일반적으로" 또는 "C++에서" 사용.

## 스타일

### 분량

- **단순 주제**: 30~50줄. 목록과 코드 블록 위주.
- **복잡한 주제**: 100~160줄. 섹션 6개 이하.
- **금지**: 장황한 서론, 반복 설명, 무관한 내용.

### 구조

| 순서 | 내용 |
|------|------|
| 서론 | 주제를 한 문장으로 정의. 1~2줄. |
| 핵심 | 가장 중요한 내용 먼저. 코드/명령어 상단 배치. |
| 상세 | 표나 코드 블록으로 압축. 필요할 때만. |
| 비교 | "올바른 vs 피해야 할", "흔한 실수" 섹션. |
| 요약 | bullet point로 핵심 정리. |

### 문장

- 한 문장에 하나의 정볼만.
- 글로 3줄 설명할 것을 코드 1줄로 대체.
- 비교/옵션/상황별 명령어는 무조건 표로.
- 질문 형태 제목 금지. 명사구 제목 사용.

### 코드 블록

- 실행 가능한 최소 코드.
- 주석은 what(무엇) 설명, not why(왜).
- `#include`나 네임스페이스는 복사해서 바로 쓸 수 있게 생략하지 않는다.

### 금지 사항

- ❌ IDE 설정이 메인이 되는 문서
- ❌ 번역체 ("상세히 알아보겠습니다")
- ❌ 한 섹션 40줄 이상
- ❌ 목차 8개 이상
