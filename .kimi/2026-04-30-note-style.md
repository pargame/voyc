# Note Writing Style (2026-04-30)

`voyc/notes/` 내 학습용 문서를 작성할 때는 다음 취향을 반영한다.

## 1. 분량

- **단순 주제** (짧은 개념, 명령어 모음 등): 30~50줄. 목록과 코드 블록 위주.
- **복잡한 주제** (언어 기능, 설계 패턴 등): 100~160줄. 섹션으로 나누되 6개 이상 넘어가지 않는다.
- **절대 금지**: 장황한 서론, 반복 설명, 주제와 무관한 내용.

## 2. 구조

| 순서 | 내용 |
|------|------|
| 서론 | 주제를 한 문장으로 정의. 왜 이 문서를 보는지를 1~2줄로. |
| 핵심 | 가장 중요한 내용을 먼저. 코드 예제나 명령어를 상단에 배치. |
| 상세 | 필요할 때만 펼쳐보는 추가 설명. 표나 코드 블록으로 압축. |
| 비교 | "올바른 방법 vs 피해야 할 방법", "흔한 실수" 섹션을 넣는다. |
| 요약 | 마지막에 bullet point로 핵심만 정리. |

## 3. 문장 스타일

- **설명은 짧게**: 한 문장에 하나의 정볼만. 여러 줄로 쪼개는 것을 두려워하지 않는다.
- **코드로 설명**: 글로 3줄 설명할 것을 코드 1줄로 대체한다.
- **표로 정리**: 비교, 옵션, 상황별 명령어는 무조건 표로.
- **질문 형태 금지**: "왜 이런 구조인가?" 같은 질문은 금지. 대신 "중첩 폴�더 구조의 이유"처럼 명사구 제목을 쓴다.

## 4. 코드 블록

- 예제는 **실제로 실행 가능한 최소 코드**를 보여준다.
- 주석은 왜(why)가 아니라 무엇(what)을 설명한다.
- 불필요한 `#include`나 네임스페이스 선언은 생략하지 않는다 — 복사해서 바로 쓸 수 있게.

## 5. "왜"에 대한 설명

- 디자인 결정의 이유는 간결히. 2~3줄을 넘지 않는다.
- 예: "`voyc/` 디렉터리는 헤더 이름 충돌을 방지하고 패키지 식별자 역할을 한다."

## 6. 금지 사항

- ❌ IDE 설정이 메인이 되는 문서 (별도 파일로 분리)
- ❌ "상세히 알아보겠습니다" 같은 번역체
- ❌ 한 섹션이 40줄 이상
- ❌ 목차가 8개 이상
